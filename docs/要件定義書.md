# システム要件定義書

## システム概要
マークダウンでの執筆機能を備えた、個人用のブログおよびコンテンツ管理システム（CMS）です。
Next.jsのApp Routerによる高速なレンダリングと、Supabaseを利用したセキュアなバックエンドを構築します。

## 機能要件

### 共通項目
* ヘッダーおよびフッターの表示
  * サイトタイトル「NINGEN GAKUSHU NOTE」（公開画面・管理画面共通）
  * コピーライト「© 2026 NINGEN GAKUSHU NOTE.」（公開画面・管理画面共通）
* 画面間のナビゲーション導線
  * 公開画面から管理画面へのリンク（管理者ログイン）
  * ログイン画面から公開画面への戻る導線（公開サイトへ戻る）
  * 管理画面右上から公開画面へのプレビュー導線（公開サイトを確認）※新しいタブで開く
  * 管理画面右上にログアウトリンクを配置
  * 記事詳細画面から一覧への戻る導線（一覧に戻る）

### 公開画面 (Public)
* 記事一覧表示（すべての記事、およびタグ別の絞り込み表示。タイトル、タグ、投稿日時、更新日時をカード型で表示）
  * タグフィルターは、記事に紐づくタグから動的に生成する。先頭に「すべて」ボタンを配置。
  * ※更新日時は、投稿日時と異なる（後日更新されている）場合のみ表示する。
* 記事詳細表示（タグ・投稿日時・更新日時などのメタデータと、Markdown本文のHTMLレンダリング）

### 管理画面 (Admin)
* ログイン画面
  * Supabase Authを用いたセキュアな管理者ログイン機能（メールアドレス＋パスワード）
  * 画面上部に説明文「管理者用ページです。」を表示
* 記事のダッシュボード表示（投稿済みの全記事リスト）
  * リスト化される項目：タイトル、タグ、投稿日、更新日、ステータス、操作（編集/削除ボタン）
  * リスト上部に「+ 新規記事を作成」の明確なリンクを配置
* 記事の作成・編集機能（同一画面で処理。編集時はURLパラメータでIDを受け取る）
  * 入力項目：タイトル、URLスラグ、投稿日、更新日、タグ、Markdown本文
  * ステータス切り替え（下書き / 公開）は、ラジオボタンで実装
  * ※投稿日・更新日は任意で手動変更可能。未指定時は保存・更新時の日時に自動設定。
  * ※タグはカンマ区切りで入力することで、複数指定が可能。
  * 画面下部に「保存する」ボタンと「戻る」リンク（ダッシュボードへ遷移）を配置
* 記事の削除機能
  * ダッシュボードの「削除」ボタン押下時、確認ダイアログ（「本当に削除しますか？」）を表示してから実行

## 非機能要件・技術スタック

* フロントエンド: Next.js (App Router), TypeScript
* データベース: Supabase (PostgreSQL)
* ORM: Prisma
* セキュリティ: Supabase Authによる認証と、Row Level Security (RLS) によるデータアクセス制御
